# ADR-0004: Authentication — JWT (access + refresh, hashed refresh storage)

**Status:** Accepted (2026-06-27)
**Phase:** 0 (used starting Phase 1)

## Context

- Frontend and backend are separate concerns; eventual multi-origin deployment is plausible.
- The public mock endpoint `/mock/:slug/*path` benefits from optional request identity in logs (not enforced auth).
- The user wants to demonstrate modern auth patterns: short-lived access, longer refresh, rotation.

## Decision

Use **stateless JWT**:
- Access token: 15 min TTL, secret `JWT_ACCESS_SECRET`
- Refresh token: 7 day TTL, secret `JWT_REFRESH_SECRET`
- Refresh tokens are **hashed (bcrypt)** and stored in `refresh_tokens` table with `expires_at`, `revoked_at`, `ip`, `user_agent`
- Refresh rotation: every refresh issues a new access + new refresh; the old refresh is marked revoked
- Claims: `sub` (user id), `email`, `role` (global), `projects[]` (per-project roles cached at login)
- Storage on the client: `localStorage` (Phase 1); migrate to `httpOnly` cookies in Phase 5

## Consequences

**Positive**
- Stateless backend; horizontally scalable.
- Refresh rotation reduces blast radius if a refresh token leaks.
- Claims carry per-project role → RBAC decisions local to the token, no DB roundtrip on hot path.

**Negative**
- Cannot instantly revoke an access token. Mitigation: short TTL + refresh rotation.
- `localStorage` is XSS-vulnerable. Documented as future work (httpOnly cookies + CSRF in Phase 5).
- Logout is best-effort: client deletes tokens; server cannot enforce until refresh expires.

## Alternatives considered

- **Session cookies + Redis store** — simpler revocation but requires CSRF protection and cookie-domain strategy. Rejected for the first cut.
- **Paseto instead of JWT** — better defaults but smaller community; not worth the friction for a portfolio project.

## Notes

- All auth-related middleware lives in `backend/internal/middleware/auth.go` and `backend/internal/middleware/rbac.go`.
- Password hashing uses `bcrypt` cost 12; tunable via `BCRYPT_COST` env.
